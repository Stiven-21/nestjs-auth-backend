# Support

## NEST AUTH Support

Thank you for using **NEST AUTH**.

This document explains how to get help, report issues, and request support in a clear and efficient way.

## üìå Scope of Support

NEST AUTH is an **open-source project** maintained by the community.

Support is provided on a **best-effort basis** and covers:

- Usage questions
- Configuration guidance
- Clarifications about documentation
- Bug reports
- Security-related concerns (see Security section below)

The following are **out of scope**:

- Custom feature development
- Private consulting or SLA-based support
- Debugging proprietary or heavily modified forks

---

## üêõ Reporting Bugs

If you believe you have found a bug:

1. Search existing issues to avoid duplicates
2. Open a new issue in the GitHub repository
3. Provide as much detail as possible

Please include:

- NEST AUTH version
- Node.js version
- Database type and version
- Steps to reproduce
- Expected vs actual behavior
- Relevant logs or error messages (sanitized)

## üí° Feature Requests

Feature requests are welcome and encouraged.

- Open a GitHub issue

Well-structured proposals are more likely to be discussed and accepted.
